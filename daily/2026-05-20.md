## 学习笔记

### 关键概念

**Chain-aware Context 核心思想**：把链上事实变成模型可读、可引用、可验证的上下文。

**七大知识节点**：
| 节点 | 难度 | 一句话 |
|------|------|--------|
| On-chain Data | 初级 | 余额、交易、日志、合约状态等可验证数据 |
| Contract Docs | 初级 | ABI 不够，还需 NatSpec、README、审计报告 |
| ABI / Event | 中级 | 函数签名 + 业务日志（Transfer, Swap, Deposit） |
| Transaction History | 中级 | 用户/合约的历史行为，需保留 tx hash 等证据 |
| Explorer Context | 初级 | 区块浏览器链接作为可核验入口 |
| Indexing Context | 中级 | 事件 → 可查询数据，必须带时间戳和同步状态 |
| Citation | 初级 | 每条链上结论必须能追溯到具体证据 |

**核心原则**：
- 模型不能凭语言记忆判断链上事实
- 链上状态有时间性，上下文必须带来源
- 区分「事实」（工具返回）和「解释」（模型输出）
- 能调用 ≠ 应该调用（写交易前需权限/余额/simulation 检查）
